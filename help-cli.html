CLI · 构建 · 发布流水线

核心命令

  • jvav init <name>：创建项目骨架，包含脑波适配层。
  • jvav build：打包解释型脚本，附带哈希与签名信息。
  • jvav run [file]：运行 JVAV 项目或指定文件，支持自动检测 main.jvav 或 .jvavpkg 文件。
  • jvav verify <file.jvavpkg>：校验包完整性与签名，确保可重复验证。
  • jvav calibrate：进入脑波校准模式，生成基线数据。
  • jvav.exe -f <file.jvav>：直接从文件运行脚本，支持 for 循环等。

构建流水线示例

在 CI 中可以串联：校验 → 打包解释器 → 验证 → 签名 → 发布。伪流程：

jvav verify deps.jvavpkg
jvav build
jvav verify dist/app.jvavpkg
sign dist/app.jvavpkg > dist/app.jvavpkg.sig
publish dist/app.jvavpkg dist/app.jvavpkg.sig

环境与缓存

CLI 默认将缓存放在 %LOCALAPPDATA%/JVAV/cache，可通过环境变量 JVAV_CACHE 指定目录。CI 中可启用缓存加速构建。

集成到现有项目

你可以仅使用 JVAV 的脑波输入层，把指令输出到已有后端：解释器直接驱动后端流程。通过 jvav run --pipe 将指令序列输出到标准输出，再由你的系统消费。

调试技巧

  • 开启 JVAV_LOG=debug 查看编译与信号处理细节。
  • 使用 jvav run --trace 输出指令转换链路。
  • 遇到延迟：关闭非必要插件，确认本地防病毒未拦截 CLI。

扩展函数支持

解释器现支持更多倒序函数，如 mus（求和）、xam（最大值）、nioj（连接）等，便于复杂逻辑表达。
